# 重复报盘检测逻辑

## 术语定义

| 术语 | 含义 |
|------|------|
| 源文件 | 银行报盘目录下的 `.xls` 文件，格式如 `YYYYMM-银行-单位名.xls` |
| 子组 | 在同一个大单位/单位名下，按合并策略进一步细分的文件集合。同一个子组内的文件最终合并为**同一个输出 Excel**。子组的划分逻辑见下方详解 |
| 大单位 | 按映射规则归类后的单位名称（如多个下属单位归到同一集团公司） |
| 排除项 | 映射规则中标为"排除"的单位，不按大单位合并，各自独立输出 |
| `filtered_reason` | 每条操作记录中的字段，标记该行被过滤的原因 |
| `dup_warning` | 标记该行数据存在重复可能 |

## 子组（Sub Group）详解

### 概念

子组是合并流程中的基本工作单元。每个子组内的所有有效数据最终被写入**同一个输出 Excel 文件**。

同一个子组内包含的是：**同大单位、同年月（或同银行）、同银行（或同年月）** 的源文件集合。

### 划分流程（两步）

**第一步：按单位归类**（确定哪些文件在同一个大组里）

```
文件列表 → 解析每个文件的 (yearmon, bank, unit_name)
        → get_big_org(unit_name) 得到大单位名
        → is_excluded(unit_name) 判断是否排除项
        → 非排除项：group_key = big_org
        → 排除项：group_key = unit_name（原始单位名，各自独立）
        → 拥有相同 group_key 的文件归入同一个组
```

**第二步：按年月+银行拆分子组**（确定这个组里拆成几个输出文件）

```
merge_different_banks = False, merge_different_months = False
    → 年月份|银行 都不同的拆到不同子组（默认）
    → 例：(202606, 吉林银行) 一个子组，(202606, 建设银行) 另一个子组

merge_different_months = True, merge_different_banks = False
    → 按银行拆，同银行不同月份合在一起
    → 例：202606-吉林银行 + 202607-吉林银行 → 同一个子组

merge_different_banks = True, merge_different_months = False
    → 按月份拆，同月份不同银行合在一起
    → 例：202606-吉林银行 + 202606-建设银行 → 同一个子组

merge_different_banks = True, merge_different_months = True
    → 全部合到一个子组
```

### 举例

假设有以下文件（映射规则中吉林大学第二医院 → 大单位 = 吉林大学第二医院）：

```
202606-吉林银行-吉林大学第二医院(全风险劳务派遣).xls
202606-吉林银行-吉林大学第二医院(全风险劳务派遣)(1).xls
202606-吉林银行-吉林大学总院.xls
202607-吉林银行-吉林大学第二医院(全风险劳务派遣).xls
```

**默认合并策略（按年月+银行拆分，不合并）**：

| 大单位 | 子组 key | 包含文件 |
|--------|---------|---------|
| 吉林大学第二医院 | `202606\|吉林银行` | 吉林大学第二医院(全风险劳务派遣).xls、吉林大学第二医院(全风险劳务派遣)(1).xls |
| 吉林大学总院 | `202606\|吉林银行` | 吉林大学总院.xls |
| 吉林大学第二医院 | `202607\|吉林银行` | 吉林大学第二医院(全风险劳务派遣).xls |

→ 输出 3 个 Excel 文件。

**开启合并不同月份**（`merge_different_months = True`）：

| 大单位 | 子组 key | 包含文件 |
|--------|---------|---------|
| 吉林大学第二医院 | `吉林银行` | (202606 + 202607) × (全风险劳务派遣 + (1)) 全部 3 个文件 |
| 吉林大学总院 | `吉林银行` | 吉林大学总院.xls |

→ 输出 2 个 Excel 文件。

### 文件级去重的子组边界

文件级去重**仅在同一子组内**比较。上例默认策略下：
- 吉林大学第二医院 202606 的子组有 2 个文件 → 会互相比较文件签名
- 吉林大学总院只有 1 个文件 → 无可比较对象
- 吉林大学第二医院 202607 只有 1 个文件 → 无可比较对象

但如果：

```
202606-建设银行-中建银(北京)房地产土地资产评估有限公司(1).xls  （排除项）
202606-建设银行-中建银(北京)房地产土地资产评估有限公司(4).xls  （排除项）
```

排除项的 `group_key = 原始单位名`，所以两个文件分属**不同组** → 各自独立成子组 → 文件级去重不会比较它们。这是引入**跨子组文件级去重**（`global_fp_cache`）的原因。

## 整体流程

```
源文件列表 → 文件级去重（同子组+跨子组） → 读取数据 → 同子组内行级判重 → 
输出合并文件 → 文件名含-可能重复判定 → 全局判重 → 文件级完全重复检测 → 
生成操作记录
```

去重分三个层次，逐层递进：

1. **文件级去重** — 内容完全相同的源文件只保留一份
2. **同子组内行级判重** — 同一输出文件内重复行，仅警告不剔除
3. **全局判重** — 跨输出文件发现重复行，仅警告不剔除

---

## 一、文件级去重

### 触发时机

每个子组读取完所有源文件数据后（`all_rows` 和 `all_operation_records` 已构建），在执行合并输出之前。

### 判定依据

对子组内每个源文件，提取其所有有效记录（金额≠0 且未被过滤），每条记录生成签名元组：

```
signature = (source_yearmon, person_or_account, account_or_person, amount)
```

- `source_yearmon`：源文件年月（如 `202606`）
- `person_or_account`：`id_number` 字段（身份证号/账号），去空格后填入；若无则填空字符串
- `account_or_person`：`name` 字段（收款户名），去空格后填入；若无则填空字符串
- `amount`：金额，转为 `float`

一个文件的完整签名为所有记录签名元组的 **frozenset**。

### 比较方式

1. **同子组比较**（本地缓存 `local_fp_cache`）：
   - 子组内所有源文件按文件名排序，依次计算 frozenset
   - 若当前文件的 frozenset 已在本地缓存中存在 → **文件级重复**
   - 缓存 key = frozenset，value = 最早出现的文件名

2. **跨子组比较**（全局缓存 `global_fp_cache`）：
   - 全局缓存 key = `big_org`（大单位），value = `{frozenset: 文件名}`
   - 每个子组处理完毕后，将其未剔除文件的 frozenset 加入全局缓存
   - 当前子组的文件也会与全局缓存中**同大单位**的已处理文件比较
   - 解决排除项（如中建银(1) 和 中建银(4)）因 `group_key` 不同而分到不同子组、无法在同一个子组内比较的问题

### 用户交互

通过 `dedup_callback(file_a, file_b) → 'skip_dup' | 'keep_both'` 回调：

| 用户操作 | 回调返回值 | 行为 |
|---------|-----------|------|
| 是（剔除） | `"skip_dup"` | 将重复文件加入 `skip_files_set`，其所有记录标记 `filtered_reason = "文件重复，已去重"`，从 `all_rows` 移除，不写入输出文件 |
| 否（保留两份） | `"keep_both"` | 保留所有记录，两条文件均写入输出 |
| 取消 | 抛异常 | 中止合并流程 |

**优先剔除带复制后缀的文件**，判定规则：

```python
stem = os.path.splitext(filename)[0]  # 去掉 .xls 后缀
normalized = _normalize_unit_name(stem)  # 去掉末尾的 (1), - 副本, - Copy 等后缀
if stem != normalized:
    # 该文件带复制后缀 → 优先建议剔除
```

### 影响

- 被剔除文件的记录：`filtered_reason = "文件重复，已去重"` → 下游所有判重逻辑跳过
- 操作记录中这些行的备注列显示"文件重复，已去重"，粉色底色
- 去重决策记录写入 `file_dedup_log`，最终输出到操作记录 Excel 的「文件去重记录」sheet

## 二、同子组内行级判重

### 触发时机

文件级去重完成后，同一子组内剩余有效记录之间。

### 判定依据

```
dup_key = (source_yearmon, big_org, person_or_account, account_or_person, amount)
```

- `source_yearmon`：源文件年月
- `big_org`：该记录归类的大单位
- `person_or_account`：`id_number` 字段（去空格）
- `account_or_person`：`name` 字段（去空格）
- `amount`：金额（`float`）

如果同一 `dup_key` 出现 **≥2 条**记录，判定为可能重复报盘。

### 处理方式

**仅警告，不剔除。** 因为无法判断用户意图 — 同一大单位下同一人同一天发两笔相同金额可能是正常的（如补发+当月工资）。

- 被标记行：`dup_warning = True`
- 生成警告："可能重复报盘：{大单位} 月份{年月} 户名「{姓名}」金额{金额}元 在{N}条记录中出现（来源：文件A、文件B）"
- 警告写入 `warnings_list`

---

## 三、全局判重

### 触发时机

所有子组处理完毕、所有输出文件写入完成后。

### 判定依据

与同子组内行级判重相同：

```
dup_key = (source_yearmon, big_org, person_or_account, account_or_person, amount)
```

扫描 **全部** `all_operation_records`（所有子组的所有记录），跳过 `filtered_reason` 不为空的记录。

### 处理方式

与同子组内行级判重相同 — **仅警告，不剔除**。

```
可能重复报盘：{big_org} 月份{ym} 户名「{person}」金额{amount}元 在{len(recs)}条记录中出现
（来源：{file_A}、{file_B}），详见操作记录末尾
```

- 被标记行：`dup_warning = True`
- 警告写入 `warnings_list`
- 全局判重**不修改** `filtered_reason`，不影响输出内容

---

## 四、输出文件 "-可能重复" 后缀

### 触发时机

每个子组生成合并输出文件名时。

### 判定依据

```python
dup_seen = defaultdict(int)
for rec in all_operation_records:          # 所有已处理的记录
    if rec.get("filtered_reason"): continue  # 跳过已被过滤的
    k = (rec["source_yearmon"], rec["big_org"], ...)
    dup_seen[k] += 1

current_keys = set()
for rec in all_operation_records[records_before:]:  # 当前子组的记录
    if rec.get("filtered_reason"): continue
    current_keys.add(k)

has_dup = any(dup_seen[k] >= 2 for k in current_keys)
```

如果当前子组中存在任何时候出现过的 `dup_key`（含跨子组），则输出文件名追加 `-可能重复.xlsx`。

---

## 五、文件级完全重复检测（文件名后缀用）

### 触发时机

全局判重之后。

### 判定依据

对每个 `source_file`，检查其所有有效记录中 **有多少条被标记了 `dup_warning = True`**：

```
total = 该文件所有有效记录数
dup_count = 其中 dup_warning = True 的记录数
if dup_count == total and total >= 2:
    # 该文件的每一行都是重复 → 几乎可以确定是文件复制
```

当至少有两个文件的 `dup_count == total` 时，进一步比较这些文件的 frozenset 签名（同文件级去重），确认内容完全相同。

### 处理方式

仅生成警告，不修改输出内容。

---

## 六、操作记录「文件去重记录」Sheet

### 数据来源

`file_dedup_log` 列表，每次文件级去重决策时追加一条记录：

```python
{
    "dedup_file": "被剔除/匹配的文件名",
    "keep_file": "保留的文件名", 
    "action": "已剔除" | "已保留两份",
    "detail": "（可选，行级去重时的详细信息）"
}
```

### 输出列

| 列 | 内容 |
|----|------|
| 序号 | 递增序号 |
| 被剔除文件 | 被标记剔除的文件名（保留两份时为空） |
| 保留文件 | 保留的文件名 |
| 操作 | `已剔除` 或 `已保留两份` |
| 详情 | 行级去重时的明细（目前保留扩展用） |

### 与其他数据的联动

- `filtered_reason` 不为空的记录在操作记录「合并映射明细」sheet 中粉色底色显示
- `dup_warning = True` 的记录黄色底色显示
- 一条记录可能同时被 `filtered_reason` 标记（文件级去重剔除）和 `dup_warning` 标记（行级重复），此时备注显示"文件重复，已去重；可能重复报盘"，底色为粉色（`filtered_reason` 优先）

## 附录：判定逻辑流程图

```
文件读取完成
    │
    ▼
文件级去重
    ├── 同子组 local_fp_cache 比较
    ├── 跨子组 global_fp_cache 比较
    ├── 用户弹窗选择剔除/保留
    └── 被剔除 → filtered_reason 标记，移出 all_rows
    │
    ▼
同子组内行级判重
    ├── 按 (yearmon, big_org, person, account, amount) 分组
    ├── 出现 ≥2 次 → dup_warning 标记
    └── 仅警告，不剔除
    │
    ▼
输出文件命名
    ├── 扫描全部已有记录
    └── 有跨子组重复 → 文件名加 -可能重复
    │
    ▼
写入合并输出文件
    │
    ▼
全局判重
    ├── 跳过 filtered_reason 不为空的记录
    ├── 按 (yearmon, big_org, person, account, amount) 分组
    ├── 出现 ≥2 次 → dup_warning 标记
    └── 仅警告，不剔除
    │
    ▼
文件级完全重复检测
    └── 文件内所有行都被标记 → 判定为文件复制，仅警告
    │
    ▼
生成操作记录
    ├── 合并映射明细 sheet（含 filtered_reason 和 dup_warning 标记）
    ├── 汇总 sheet
    └── 文件去重记录 sheet（文件级去重决策明细）
```
